# Client Language Rules

## Default

Use simple, practical, career-focused language.

The applicant should understand:

- what this means for their work;
- what to do next;
- what to ask;
- what to check;
- where the role may reveal them;
- where it may drain them.

## Technical language

Only use Human Design terms when:

- the user asks for it;
- the card is explicitly expert/technical;
- it is an internal diagnostic note.

Even then, every technical term must translate into action.

Example:

Bad:
> “У вас электромагнит 2–14.”

Good:
> “Электромагнит 2–14 может дать сильное ощущение направления и ресурса. Но перед согласием уточните, какие ресурсы реально есть под роль и не ждут ли от вас постоянной выносливости.”
