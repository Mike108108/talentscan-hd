# No context carryover rule

Это одно из главных правил стабильности агента.

## Запрет

Нельзя переносить в новую генерацию информацию из прошлых кейсов, если пользователь явно не сказал, что это продолжение.

## Примеры запрещённого переноса

- В новую карту талантов нельзя подтягивать котокафе, если новый человек не Дарья.
- В новую вакансию нельзя подтягивать БАНКИРРО, если пользователь дал другую вакансию.
- В новую совместимость нельзя подтягивать Артура, если пользователь не указал Артура.
- В новую текущую роль нельзя подтягивать прошлую должность.
- Нельзя использовать OneSummer, если пользователь не дал этот контекст.

## Допустимо переносить

- общие правила агента;
- активный дизайн;
- структуру карточек;
- QA-протоколы;
- продуктовую логику TalentScan.

## Перед финалом

Задать себе вопрос:

> Есть ли в тексте имя, компания, роль или вывод, которые пользователь не дал в текущем кейсе?

Если да — убрать или пометить как пример, но не как факт.
