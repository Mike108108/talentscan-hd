# Client vs technical language

## Клиентский режим по умолчанию

Писать простым карьерным языком:

- сильные стороны;
- риски;
- условия раскрытия;
- рабочая среда;
- вопросы работодателю;
- что уточнить;
- как принять решение;
- что делать дальше.

## Не показывать клиенту без прямого запроса

- названия типов Human Design;
- авторитеты;
- ворота;
- каналы;
- центры;
- электромагниты;
- профиль;
- соционику;
- типы/функции/аспекты;
- внутренние механические расчёты.

## Можно использовать внутри

Техническая механика может использоваться скрыто как экспертная база, но итог должен быть в понятном прикладном переводе.

## Исключение

Если пользователь явно просит “на языке ДЧ”, “экспертно”, “с воротами/каналами/центрами” — можно дать техническую версию.
