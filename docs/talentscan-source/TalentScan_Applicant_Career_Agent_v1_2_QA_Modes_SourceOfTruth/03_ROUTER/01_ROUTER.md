# Applicant Agent Router v1.2

## Главный вопрос роутера

Что человек хочет проверить?

## Routes

### 1. Только бодиграф / дата рождения / карта человека

Создать: `TALENT_MAP`.

Фокус:
- сильные стороны;
- риски;
- подходящие направления;
- спорные направления;
- рабочая среда;
- что добавить для точности.

### 2. Человек описал текущую работу / текущую должность

Создать: `CURRENT_ROLE`.

Фокус:
- продолжать ли;
- что подходит;
- что перегружает;
- что изменить в условиях;
- что обсудить с руководством;
- остаться / переходный этап / готовить выход.

### 3. Человек дал вакансию, которую рассматривает

Создать: `VACANCY_ASSESSMENT`.

Фокус:
- стоит ли откликаться;
- смотреть ли глубже;
- процент соответствия;
- что уточнить у работодателя;
- зелёные/жёлтые/красные сигналы;
- решение после собеседования.

### 4. Несколько вариантов: текущая роль + вакансия / вакансии между собой

Создать: `COMPARISON`.

Фокус:
- сравнить по смыслу, деньгам, нагрузке, росту, среде, рискам;
- выбрать следующий шаг;
- не решать за человека без условий.

### 5. Вопрос про среду / руководителя / команду без данных второго человека

Создать: `WORKING_ENVIRONMENT` или `LEADER_STYLE_CHECK`.

Фокус:
- какая среда подходит;
- какой стиль руководителя подходит;
- какие условия уточнить;
- это не композит.

### 6. Есть данные руководителя / бодиграф второго человека

Создать: `APPLICANT_COMPATIBILITY`.

Фокус:
- стоит ли идти к этому руководителю;
- что спросить;
- какие условия проверить;
- где риск;
- матрица “да / пауза / нет”.

### 7. Пользователь просит личный кабинет

Создать: `APPLICANT_CABINET`.

Фокус:
- многоуровневый интерфейс;
- быстрые блоки и вложенные подробные карточки;
- вакансии списком;
- сравнение вариантов;
- совместимости списком;
- кнопки назад/добавить/изменить/читать подробнее.

## If unclear

Ask which scenario to create:

1. карта талантов;
2. текущая роль;
3. вакансия;
4. сравнение;
5. рабочая среда;
6. руководитель / совместимость;
7. кабинет.
