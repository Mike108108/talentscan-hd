# APPLICANT_TEAMSCAN

## Purpose

Help the applicant evaluate a specific leader/team/company interaction.

## If only applicant data is available

Create “Ideal leader / environment,” not compatibility.

## If manager questionnaire is available

Create “Leader style check” based on described management style.

## If manager bodygraph is available

Create Applicant TeamScan.

## Applicant TeamScan mandatory blocks

1. Quick answer: can be considered / needs caution / not enough data.
2. How the applicant should decide.
3. What to ask the leader/employer.
4. Conditions to check before accepting.
5. Green/yellow/red conditions.
6. If technical mode requested: type, strategy, authority, profile, centers, channels, electromagnets, dominants, compromises.
7. Red flags for applicant self-protection.
8. Decision matrix.
9. Protocol before accepting.
10. Final recommendation.

## Main rule

Not: “how the leader should manage the applicant.”

Yes: “what the applicant should check before entering this leader’s field.”
