# General QA before output

Перед выдачей любой карточки агент обязан проверить:

## 1. Сценарий

- Правильно ли определён тип карточки?
- Это карта талантов, текущая роль, вакансия, сравнение, среда, совместимость или кабинет?
- Если сценарий неоднозначен — уточнил ли агент?

## 2. Независимость кейса

- Не подтянуты ли данные прошлых людей?
- Не подтянута ли прошлая компания/вакансия/роль?
- Не используется ли имя Дарья, Артур, БАНКИРРО, котокафе, OneSummer как факт без данных пользователя?

## 3. Applicant framing

- Текст написан для соискателя?
- Нет ли HR-угла “работодатель решает, брать ли кандидата”?
- Есть ли фокус на: роль, среда, руководитель, условия раскрытия, вопросы, решение?

## 4. Данные

- Разделено ли: известно / гипотеза / нужно уточнить?
- Не выдуманы ли KPI, компания, руководитель, условия?
- Есть ли список уточнений при нехватке данных?

## 5. Формат

- HTML по умолчанию?
- PDF только по просьбе?
- Активный dark/darya style сохранён?

## 6. Клиентский язык

- Нет технических терминов без запроса?
- Нет перегруза методологией?
- Есть понятные действия и вопросы?

## 7. QA по типу карточки

Выполнить отдельный QA-файл из этой папки.
