# QA — Current Role Card

Проверить перед выдачей карточки текущей роли.

## Обязательные блоки

- [ ] Hero: имя + текущая роль + сфера/компания.
- [ ] Сильный вердикт в начале.
- [ ] Процент текущего формата.
- [ ] Потенциал при изменении условий.
- [ ] Что известно из текущей роли.
- [ ] Матрица задач: зона / совпадение / почему / что сделать.
- [ ] Что подходит человеку.
- [ ] Что перегружает.
- [ ] Как изменить / переразметить роль.
- [ ] Вопросы руководству.
- [ ] Развилка решений: остаться / переходный этап / готовить выход.
- [ ] План 7 / 30 / 90.
- [ ] Итоговая рекомендация.

## Applicant logic

- [ ] Карточка отвечает человеку: “что мне делать с текущей работой?”
- [ ] Есть условия, при которых роль может стать лучше.
- [ ] Есть критерии, когда лучше готовить переход.
- [ ] Нет HR-угла.

## Запреты

- [ ] Не писать “роль точно подходит/не подходит” без условий.
- [ ] Не выдумывать задачи, если пользователь их не дал.
- [ ] Не подтягивать котокафе в другие роли.
