# QA — Vacancy Assessment Card

Проверить перед выдачей оценки вакансии.

## Обязательные блоки

- [ ] Hero: имя + название вакансии + компания.
- [ ] Сильный быстрый вердикт.
- [ ] Процент общего соответствия.
- [ ] Краткие данные вакансии: зарплата, график, формат, обязанности, требования.
- [ ] Главные плюсы вакансии.
- [ ] Главные риски.
- [ ] Матрица задач: зона / совпадение / почему / что проверить.
- [ ] Что уточнить у работодателя.
- [ ] Зелёные, жёлтые, красные сигналы.
- [ ] Блок “Как преподнести себя на собеседовании” / “Как себя продать под вакансию”.
- [ ] В блоке самопрезентации есть: главная позиция, что подчеркнуть, как говорить про опыт, какие риски закрыть, что не выпячивать, готовые фразы.
- [ ] Матрица решения после собеседования.
- [ ] План: до отклика / на интервью / после интервью.
- [ ] Итоговая рекомендация.

## Applicant logic

- [ ] Карточка написана как “я рассматриваю вакансию”.
- [ ] Есть вопрос: стоит ли откликаться / смотреть глубже / взять паузу.
- [ ] Есть список вопросов работодателю.
- [ ] Есть проверка условий, а не только оценка “подходит”.

## Запреты

- [ ] Не делать финальный приговор без ответов работодателя.
- [ ] Не выдумывать KPI и стиль руководства.
- [ ] Не писать, что компания “точно такая-то”, если это не из вакансии.
- [ ] Не подтягивать прошлую вакансию.
