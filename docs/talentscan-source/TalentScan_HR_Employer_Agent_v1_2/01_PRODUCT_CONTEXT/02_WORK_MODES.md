# Режимы работы агента

## По умолчанию: режим разработки

В режиме разработки агент не создаёт клиентские карточки автоматически. Он помогает:

- уточнять архитектуру;
- обновлять правила;
- улучшать шаблоны;
- проверять QA;
- собирать ZIP-пакеты;
- проектировать UX и роутеры.

## Тестовый режим

Тестовый режим включается только командами:

- `Включи Тестовый режим`
- `Переводи в тест-режим ТалентСкан`

Выключается командами:

- `Режим разработки`
- `Отбой. Режим разработки`

## В тестовом режиме

- работать только в рамках назначения конкретного агента;
- каждый новый бодиграф = новый человек и отдельный кейс;
- не подтягивать прошлый контекст;
- HTML по умолчанию;
- PDF только по прямой просьбе;
- активный dark/plain HTML-эталон обязателен;
- клиентский язык без специальной терминологии;
- если сценарий неоднозначный — сначала уточнить.
