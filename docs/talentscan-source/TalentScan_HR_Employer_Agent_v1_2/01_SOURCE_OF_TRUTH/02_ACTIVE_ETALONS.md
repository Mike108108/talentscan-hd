## Активные клиентские HR-эталоны v1.2

### 0. Карта талантов человека

`11_HTML_EXAMPLES/ACTIVE_HR_CLIENT_ETALONS_v1_2/ETALON_00_HR_PERSON_TALENT_MAP_DARYA_v1_1.html`

Использовать для сценариев:
- человек без привязки к конкретной вакансии;
- кандидат как отдельная рабочая личность;
- сотрудник;
- руководитель;
- человек без роли;
- базовый профиль перед оценкой под вакансию.

Обязательные блоки:
- общий вывод;
- показатели оценки;
- таланты в рабочем переводе;
- сильные стороны;
- риски;
- подходящие направления;
- спорные направления;
- должности;
- рабочая среда;
- проверка;
- итоговая рекомендация.

### 1. Кандидат под конкретную роль

`11_HTML_EXAMPLES/ACTIVE_HR_CLIENT_ETALONS_v1_2/ETALON_01_HR_CANDIDATE_FOR_ROLE_MARINA_ONESUMMER_SALES_v1_2_1.html`

Использовать для сценариев:
- один кандидат;
- есть роль / вакансия;
- есть компания или бизнес-контекст;
- работодатель хочет понять, вести ли кандидата дальше.

### 2. Сравнение кандидатов на одну роль

`11_HTML_EXAMPLES/ACTIVE_HR_CLIENT_ETALONS_v1_2/ETALON_02_HR_CANDIDATE_COMPARISON_MARINA_ZARINA_ONESUMMER_SALES_v1_1.html`

Использовать для сценариев:
- 2–5 кандидатов;
- одна роль / вакансия;
- нужно выбрать приоритет, финалистов, резерв и зоны проверки.

### 3. Рабочая связка руководитель × кандидат / сотрудник

`11_HTML_EXAMPLES/ACTIVE_HR_CLIENT_ETALONS_v1_2/ETALON_03_HR_TEAMSCAN_ARTUR_MARINA_ONESUMMER_SALES_v1_0.html`

## Правило клиентского слоя

Клиентские HTML-карточки не должны содержать служебные блоки:
- правила агента;
- внутреннюю методологию;
- технические термины ДЧ/соционики;
- комментарии “почему это эталон”.

Такие правила хранить только в файлах агента.
