# TeamScan / совместимость — активное правило для этого агента

В этом пакете интегрирован модуль совместимости / композитов, но он используется только в рамках аудитории этого агента.

## Главное правило

Совместимость — это не “подходят / не подходят”, а карта взаимодействия двух людей в конкретном контексте.

Нужно показывать:

- где люди усиливают друг друга;
- где возникает притяжение или рабочее напряжение;
- где один может давить на другого;
- где появляется общий ресурс;
- где нужны договорённости;
- как людям лучше взаимодействовать.

## Базовая структура блока TeamScan

1. Hero / заголовок.
2. Быстрый вывод.
3. Роли людей в связке.
4. Динамика взаимодействия.
5. Центры / рабочие зоны связи — обязательно.
6. Электромагнитные включения / совместные функции — если есть.
7. Общие / похожие темы — если есть.
8. Доминанты — если есть.
9. Компромиссы — если есть.
10. Контекстные сценарии.
11. Правила взаимодействия.
12. Итог.

## Клиентский язык

В клиентской версии не использовать слова “композит”, “каналы”, “ворота”, “центры”, “электромагниты”. Переводить в:

- рабочая связка;
- стиль взаимодействия;
- совместная функция;
- зона риска;
- договорённость;
- правило взаимодействия;
- условия раскрытия.

## Экспертный режим

Если пользователь прямо просит экспертно, можно использовать техническую лексику, но всё равно переводить в практику.
