# Общий QA-протокол перед выдачей результата

Перед выдачей HTML-карточки агент обязан проверить:

## 1. Роутинг

- сценарий определён правильно;
- тип карточки соответствует входным данным;
- если сценарий неясен — агент уточнил, а не угадал.

## 2. Данные

- имя человека / людей указано верно;
- компания, роль, вакансия, контекст не перепутаны;
- новый бодиграф не смешан с прошлым кейсом;
- если данных мало — это явно отмечено.

## 3. Формат

- создан реальный `.html` файл;
- PDF не создан без прямой просьбы;
- не создана картинка вместо HTML;
- весь текст карточки не вставлен в чат.

## 4. Дизайн и мобильная работа

- использован активный dark/plain эталон;
- topbar / burger / nav работают;
- ссылки по разделам работают;
- кнопка “Вернуться в начало” работает;
- кнопки кликаются на телефоне через tap/touch;
- decorative layers не перехватывают клики.

## 5. Клиентский язык

- нет технических терминов методологии в клиентской версии;
- нет запрещённых блоков;
- нет медицинских, интимных, телесных и фатальных выводов;
- выводы поданы как гипотезы, условия и практические рекомендации.

## 6. Финальный ответ в чат

В финальном ответе дать:

- коротко: какой сценарий выбран;
- что PDF не делался, если не просили;
- что QA проверен;
- ссылку на HTML-файл.
