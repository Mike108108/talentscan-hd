# Decision Log — Router + QA v1.1

## 2026-05-19 — Добавлены Router и QA для HR / Employer Agent

Принято:

1. HR/Employer Agent по умолчанию создаёт клиентские HTML-карточки.
2. Клиентские HTML-карточки не должны содержать служебные блоки агента, QA, эталонов и правил генерации.
3. Компания и вакансия внутри карточки должны быть компактными: работодатель уже знает свой бизнес и ЛК содержит подробные данные.
4. Главный фокус карточки — оценка кандидата под роль, сравнение кандидатов или рабочая связка руководитель × кандидат.
5. Роутер разделяет сценарии: один кандидат под роль, сравнение кандидатов, TeamScan, текущий сотрудник, профиль роли без кандидата.
6. QA обязателен перед каждой выдачей HTML: якоря, мобильная адаптация, отсутствие наложений, отсутствие служебных блоков, клиентский HR-язык, практическое решение.
7. TeamScan используется как инструмент управления и адаптации, а не как самостоятельный вердикт “брать / не брать”.
