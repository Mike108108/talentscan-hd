# HR / Employer Cabinet Architecture v1.2

## Назначение кабинета

HR-кабинет TalentScan — это рабочее пространство работодателя для управления компаниями, людьми, вакансиями, кандидатами, сравнениями, TeamScan, интервью, тестовыми, адаптацией и качеством данных.

Это не кабинет самопознания и не кабинет соискателя. Пользователь кабинета — HR, предприниматель, руководитель или владелец бизнеса.

## Основной пользовательский маршрут

```text
Выбор компании
→ Рабочий стол компании
→ Люди / Вакансии / Кандидаты
→ Карта талантов человека
→ Оценка под вакансию
→ Сравнение кандидатов
→ TeamScan с руководителем
→ Интервью / Тестовое
→ Решение
→ Адаптация
```

## Первый экран

Первый экран до входа в компанию:
- верхняя панель TalentScan / HR Employer;
- вдохновляющая фраза для HR;
- карточка OneSummer;
- кнопка “Добавить компанию”.

На первом экране не должно быть левого меню, списка людей, вакансий или лишних блоков.

## Внутри компании

После выбора компании открывается рабочий кабинет работодателя:
- верхнее меню с TalentScan / HR Employer;
- кнопка “Выбрать компанию”, которая возвращает на первый экран;
- левое меню;
- рабочий стол компании;
- основные разделы.

## Левое меню

В меню не должно быть пунктов “OneSummer” и “Компании”, потому что выбор компании вынесен на стартовый экран. Меню должно быть компактным и помещаться без лишней прокрутки.

Рекомендуемые пункты:
1. Обзор
2. Люди
3. Вакансии
4. Кандидаты
5. Сравнения
6. TeamScan
7. Интервью и тесты
8. Адаптация
9. Данные

## Обзор компании

Рабочий стол показывает:
- название компании;
- количество людей;
- количество вакансий;
- количество кандидатов;
- количество TeamScan-связок;
- блок “Что требует внимания”;
- быстрые действия: + кандидат, + вакансия, + TeamScan, активные процессы.

## Люди

Раздел “Люди” должен поддерживать фильтры:
- Все;
- Руководители;
- Кандидаты;
- Сотрудники.

Человек не должен теряться внутри вакансии. Он существует в общей базе людей и может быть привязан к одной или нескольким ролям.

## Вакансии

Во вкладке “Вакансии” должна быть возможность провалиться внутрь вакансии. Внутри вакансии должны быть:
- краткая информация по роли;
- кандидаты с процентами соответствия;
- короткие выводы по кандидатам;
- кнопки открыть кандидата / полная карточка / TeamScan;
- кнопка “Добавить кандидата”.

## Кандидаты

Во вкладке “Кандидаты” первый уровень — простой список:
- Марина — кандидат на менеджера по продажам;
- Зарина — кандидат на менеджера по продажам.

У каждого — кнопка “Смотреть подробнее”.

Внутри кандидата должны быть:
- процент соответствия;
- краткий HR-вывод;
- сильные стороны;
- риски;
- сигналы риска;
- привязки к вакансии, сравнению и TeamScan;
- кнопка “Смотреть полную карточку”;
- кнопка TeamScan;
- кнопка “Дополнить информацию”.

## Сравнение кандидатов

В разделе “Кандидаты” должна быть кнопка “Сравнить кандидатов”.
После клика HR выбирает кандидатов из списка и открывает сравнение.

## TeamScan

TeamScan — отдельный раздел рабочих связок. Связка должна появляться:
- в разделе TeamScan;
- у обоих людей;
- внутри вакансии, если связана с ролью.

## Добавление данных рождения

При добавлении кандидата, сотрудника или человека обязательно предусмотреть поля:
- дата рождения;
- город рождения;
- время рождения.
