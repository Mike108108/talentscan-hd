# HR Cabinet UX Rules v1.2

## Правило “Назад”

У каждого детального экрана должна быть контекстная кнопка “назад”.

Примеры:
- из кандидата → назад к списку кандидатов или вакансии;
- из вакансии → назад к списку вакансий;
- из TeamScan → назад к человеку, вакансии или разделу TeamScan;
- из кабинета компании → “Выбрать компанию”.

## Правило “Добавить”

В каждом рабочем разделе должна быть кнопка добавления нужного объекта:
- + компания;
- + вакансия;
- + кандидат;
- + сотрудник;
- + человек;
- + TeamScan;
- + сравнение;
- + вопрос интервью;
- + тестовое;
- + план адаптации.

## Правило автоматических связей

Если объект добавлен в одном месте, он появляется во всех связанных разделах.

Примеры:
- кандидат добавлен в вакансию → появляется в “Люди”, “Кандидаты” и внутри вакансии;
- вакансия добавлена в компанию → появляется в “Вакансии” и на dashboard компании;
- TeamScan создан между Артуром и Мариной → появляется у Артура, у Марины, в вакансии и в разделе TeamScan;
- тестовое создано для кандидата → появляется у кандидата, в вакансии и в “Интервью и тесты”.

## Защита от дублей

Если человек уже есть в базе, система должна предлагать привязать существующего человека к новой вакансии, а не создавать дубль.

## Короткое и подробное описание

В списках должны быть короткие карточки. Углубление происходит по кнопке “Смотреть подробнее”.

У кандидата должно быть два уровня:
1. короткий список “Марина — кандидат на менеджера по продажам”;
2. подробный экран кандидата с процентами, сильными сторонами, рисками и кнопкой полной HTML-карточки.

## Подробные HTML-отчёты

Подробные карточки открываются через встроенный просмотрщик:
- карта талантов человека;
- оценка кандидата под вакансию;
- сравнение кандидатов;
- TeamScan.
