# HR Cabinet Data Model v1.2

## Основные сущности

```text
Company
Person
Vacancy / Role
Candidate Application
Person Talent Map
Role Fit Assessment
Candidate Comparison
TeamScan
Interview
Test Task
Adaptation Plan
Data Completeness
Decision / Status
Note
```

## Company

Компания хранит:
- название;
- описание;
- людей;
- вакансии;
- кандидатов;
- сравнения;
- TeamScan;
- интервью;
- тестовые;
- планы адаптации;
- метрики полноты данных.

## Person

Человек хранит:
- имя;
- статус: руководитель / сотрудник / кандидат / архив;
- дата рождения;
- город рождения;
- время рождения;
- компания;
- роли и вакансии;
- карта талантов;
- оценки под роли;
- TeamScan-связки;
- интервью;
- тестовые;
- планы адаптации;
- заметки.

## Vacancy / Role

Вакансия хранит:
- название;
- компанию;
- руководителя роли;
- задачи;
- KPI;
- формат работы;
- список кандидатов;
- сравнения;
- интервью;
- тестовые;
- TeamScan;
- план адаптации;
- недостающие данные.

## Candidate Application

Связь человека и вакансии.

Один человек может быть кандидатом на несколько вакансий.

Хранит:
- человек;
- вакансия;
- статус воронки;
- процент соответствия;
- краткий вывод;
- риски;
- сильные стороны;
- тестовые;
- интервью;
- решение.

## Person Talent Map

Базовая карта человека без привязки к конкретной роли.

Используется для:
- кандидата;
- сотрудника;
- руководителя;
- человека без роли.

## Role Fit Assessment

Оценка конкретного человека под конкретную роль.

## Candidate Comparison

Сравнение 2+ кандидатов на одну роль.

## TeamScan

Рабочая связка двух людей в контексте компании/роли.

## Data Completeness

Отдельный слой качества данных. Система должна показывать, чего не хватает для точной оценки.
