# Функции в приложении для работодателей / HR, помимо карточек

| № | Функция | Описание |
|---:|---|---|
| 1 | Кабинет работодателя / HR | Даёт работодателю отдельное рабочее пространство, где собраны компании, люди, вакансии, кандидаты, сравнения, TeamScan, интервью, тестовые и адаптация. |
| 2 | Управление несколькими компаниями | Позволяет работодателю вести несколько бизнесов или направлений отдельно. |
| 3 | Рабочий стол компании | Показывает главную панель компании: активные вакансии, кандидатов, сотрудников, срочные решения, неполные данные, сравнения и рабочие связки. |
| 4 | Добавление новой компании | Позволяет создать новую компанию в кабинете. |
| 5 | Редактирование данных компании | Позволяет менять описание компании, сферу, формат работы, команды, руководителей, особенности найма и бизнес-контекст. |
| 6 | Единая база людей | Собирает всех людей компании в одном месте: руководителей, сотрудников, кандидатов, финалистов, архив и людей без роли. |
| 7 | Категории людей | Позволяет смотреть общий список или фильтровать людей по статусу: руководители, кандидаты, сотрудники. |
| 8 | Добавление нового кандидата | Позволяет добавить кандидата в систему. Если кандидат добавлен на вакансию, он автоматически появляется в “Люди”, “Кандидаты” и внутри вакансии. |
| 9 | Добавление нового сотрудника | Позволяет добавить действующего сотрудника компании и привязать его к роли, руководителю или команде. |
| 10 | Поля рождения при добавлении | При добавлении кандидата, сотрудника или человека есть поля: дата рождения, город рождения, время рождения. |
| 11 | Привязка человека к вакансии | Позволяет взять уже существующего человека из базы и привязать его к новой вакансии без создания дубля. |
| 12 | Защита от дублей | Если человек уже есть в базе, система предлагает привязать существующую карточку к новой роли. |
| 13 | Управление вакансиями / ролями | Позволяет создавать, редактировать и закрывать вакансии. |
| 14 | Провал внутрь вакансии | Позволяет открыть конкретную вакансию и увидеть роль, кандидатов, проценты соответствия, краткие выводы, интервью, тестовые и TeamScan. |
| 15 | Добавление кандидата на вакансию | Позволяет добавить нового или существующего кандидата прямо из вакансии. |
| 16 | Список кандидатов на вакансию | Показывает всех кандидатов на конкретную роль с процентами, статусами и кнопками углубления. |
| 17 | Список кандидатов | Показывает кандидатов списком: “Марина — кандидат на менеджера по продажам”, “Зарина — кандидат на менеджера по продажам”. |
| 18 | Подробный экран кандидата | Показывает проценты соответствия, сильные стороны, риски, сигналы риска, привязки к вакансии, сравнению и TeamScan. |
| 19 | Полная карточка кандидата | Позволяет открыть подробный HTML-отчёт кандидата под роль. |
| 20 | Карта талантов человека | Позволяет открыть базовую карту талантов кандидата, сотрудника или руководителя без обязательной привязки к вакансии. |
| 21 | Сравнение кандидатов | Позволяет сравнить двух или нескольких кандидатов на одну вакансию. |
| 22 | Выбор кандидатов для сравнения | Во вкладке кандидаты есть кнопка “Сравнить кандидатов” и выбор людей из списка. |
| 23 | TeamScan / рабочие связки | Позволяет оценивать рабочее взаимодействие: руководитель × кандидат, руководитель × сотрудник, сотрудник × сотрудник, команда. |
| 24 | Создание TeamScan | Позволяет выбрать двух людей, контекст и роль, после чего связка появляется в TeamScan и в карточках обоих людей. |
| 25 | Интервью | Помогает формировать вопросы для интервью под вакансию, кандидата, риски, мотивацию и рабочие задачи. |
| 26 | Добавление вопросов интервью | Позволяет HR добавлять свои вопросы или сохранять предложенные вопросы в карточке вакансии или кандидата. |
| 27 | Тестовые задания | Позволяет создавать тестовые под вакансию и кандидата: ролевая продажа, CRM-кейс, follow-up, возражения, приоритизация лидов. |
| 28 | Добавление тестового задания | Позволяет назначить тестовое конкретному кандидату и привязать результат к вакансии, интервью и финальному решению. |
| 29 | Адаптация 7 / 30 / 90 | Помогает создать план адаптации после найма: первые задачи, KPI, контрольные точки, риски, стиль управления и обратная связь. |
| 30 | Создание плана адаптации | Позволяет сформировать индивидуальный план адаптации для кандидата или сотрудника. |
| 31 | Статусы кандидатов | Позволяет вести кандидата по этапам: новый, изучить, интервью, тестовое, финалист, оффер, резерв, отказ, архив, нанят, адаптация. |
| 32 | История решений | Хранит, когда кандидат был добавлен, на какую роль, какие оценки прошёл, какие интервью и тестовые были назначены, какое решение принял HR. |
| 33 | Заметки HR | Позволяет HR и руководителю оставлять комментарии по кандидату, вакансии, интервью, тестовому, TeamScan или адаптации. |
| 34 | Полнота данных | Показывает, чего не хватает для точной оценки: KPI роли, цикл сделки, доля холодных/тёплых лидов, данные руководителя, резюме, опыт. |
| 35 | Фильтры и поиск | Позволяет находить людей, вакансии и решения по статусу. |
| 36 | Контекстные кнопки “Назад” | Помогают HR не теряться: из кандидата можно вернуться к вакансии, из TeamScan — к человеку, из сравнения — к роли. |
| 37 | Контекстные кнопки “Добавить” | В каждом разделе есть нужное действие: добавить кандидата, вакансию, сотрудника, TeamScan, интервью, тестовое, план адаптации или компанию. |
| 38 | Автоматические связи между разделами | Если кандидат добавлен на вакансию, он появляется в “Люди”; если создан TeamScan, он появляется у обоих людей; если добавлена вакансия, она появляется в компании. |
| 39 | Dashboard срочных решений | Показывает, что требует внимания: кандидаты без решения, вакансии без KPI, тестовые без проверки, TeamScan без просмотра, люди без роли. |
| 40 | Архив | Позволяет хранить закрытые вакансии, отказанных кандидатов, старые сравнения и завершённые TeamScan без удаления истории. |
| 41 | HTML-отчёты | Позволяет открывать готовые HTML-карточки по кандидату, карте талантов, сравнению, TeamScan, адаптации или финальному HR-решению. |
